问题篓子 · 机器人篇 № 002

机器人的技能，能拼起来吗？

每学一个新动作几乎都要从头训练。把"别人单独学过的技能"直接组装成一个全能选手，到底做到了什么程度。

2026-07-17 · 覆盖 arXiv / ICRA / RSS / CoRL / ICLR / SIGGRAPH / NeurIPS / GitHub · 三条整合路线
结论先行：把多个技能"整合"到一个机器人身上，目前有三条互不相同的路，成熟度差得很远——
  1. 蒸馏派（塞进一个大脑）：把多个专家蒸馏进单一网络，2024 年起已能零样本迁移到 Unitree H1/G1 真机。代价是加新技能通常要重训。
  2. 组合派（免重训拼装）：让独立学的技能在推理时直接组合。扩散策略可把动作分布相加（PoCo 已上真机），逻辑代数版本还有理论最优性——但多数停在仿真，或要求技能"不打架"。
  3. 大数据派（一次学完）：干脆不组合，用海量跨机器人数据训一个 VLA 大模型吞下几百种技能。这条路工业界已反超学界。
真正的空白是：真机上"可自由组合"仍不存在——能顺序拼接切换（冻结专家 + 门控），但"同时执行两个强耦合技能"（边走位边调整踢球弧线）仍需重训或残差微调。

先说清楚问题出在哪。你花几天训出一个会走路的策略，再花几天训一个会搬箱子的策略——按常理，会走又会搬，就该会"边走边搬"。但机器人不这么工作。把两个网络的权重简单拼在一起，得到的通常是一个既不会走也不会搬的废物；想要"走着搬"，标准做法是把两件事重新放进一个奖励函数里，从头再训一遍。技能之间没有加法，只有重来。这背后是神经网络的老毛病灾难性遗忘：学新的会覆盖旧的。于是"机器人每个技能都得单独学"就成了一句行业黑话，而"能不能把学过的拼起来"是过去几年被反复进攻、却始终没有干净答案的问题。

把这几年的工作摊开看，会发现研究者其实在用三种完全不同的哲学回答同一个问题。它们的分歧不在技巧，而在"整合"到底意味着什么。

路线 A · 蒸馏进一个网络先分头训专家，再压成单一策略。部署简单、真机主流；但技能库固定，加新的要重训。
路线 B · 蒸馏出一个技能空间把技能压进可复用的隐空间，下游冻结复用。优雅、支持组合；但几乎只活在仿真里。
路线 C · 免重训直接组合让独立策略在推理时相加/取逻辑/合并参数。真正回应"拼起来"；但有强前提，真机案例稀少。

（还有一条"作弊"路线 D：不组合，用大数据把所有技能一起学进一个基础模型。它绕开了问题，却是工业界当前的主战场，放在最后说。）

路线 A：把一群专家，蒸馏成一个大脑

这条路的思路最直接：既然一个网络同时学很多技能会互相打架，那就分开训、再合并。每个技能单独训一个专家策略（这一步容易收敛），然后用知识蒸馏——让一个学生网络去模仿所有专家的输出——把它们压进同一个网络。学生学的不是"标准答案"，而是每个专家在各种状态下的完整动作分布，因此能把多位专家的本事装进一副身体。

这个想法本身不新。早在 2019 年，DeepMind 的 Josh Merel 团队就把 2707 个单独训练的动捕模仿专家，离线蒸馏进一个带隐变量瓶颈的网络（NPMP），这也是上一期传球报告里 DeepMind 仿真球员的底层。但真正让"多专家蒸馏"在真机人形上跑起来，是 2024 年之后的事，而且几乎清一色由英伟达和一批华人实验室推动。

最具代表性的是英伟达主导、何泰然（Tairan He）一作的 HOVER。它先训一个"什么动作都会模仿"的教师，再用全身控制的多种指令模式——有的按关节角度跟踪、有的按根节点速度跟踪、有的做上肢操作——统统蒸馏进一个学生策略。结果这一个网络在 12 项跟踪指标里有 7 项以上反超各自的专用策略，仿真存活率 99.1%，还能零样本迁移到 19 自由度的 Unitree H1 真机上（靠的是训练时的动作重定向与域随机化）。一个网络、多种控制模式、直接上真机——这是"蒸馏派"最漂亮的样板。

如果说 HOVER 展示了"多模式统一"，那么北京大学与 BeingBeyond 团队的 BumbleBee（论文名直白地叫《From Experts to a Generalist》）则把"从专家到通才"的流程做成了教科书：先用自编码器给上万条动作做聚类分组，每组训一个专家、还用真机数据做增量微调，最后把这些专家蒸馏成一个统一的通才策略。在 Isaac Gym 里它的成功率 89.6%，压过 OmniH2O、ExBody2 和 HOVER。但这篇论文最有价值的其实是一张"打脸表"：当把这些方法搬到另一个仿真器 MuJoCo 上评测，HOVER 的成功率从 63% 骤降到 16%——暴露出这条路线一个被普遍回避的软肋：跨仿真器的泛化极其脆弱，很多漂亮数字换个物理引擎就崩。

还有一种更"轻"的蒸馏，不是合并多个专家，而是特权蒸馏：让一个能在仿真里"作弊"（看得到地形真值、精确状态）的教师先学会高难动作，再把它蒸馏进一个只靠真机传感器的学生。UCSD 项晓龙组的 ExBody2 是这一路的代表——教师产出运动学上可行的过滤数据，学生把精确的全身模仿技能学过来，跑跳舞蹈都能在 Unitree G1/H1 上高保真复现。它严格说不算"多技能组合"，但它和上面两者共享同一个内核：技能的获取和技能的部署，由两个不同的网络分工完成，蒸馏是它们之间的桥。

路线 A 的账很清楚：部署简单、单次前向推理、真机友好，所以真机人形几乎全走这条。但它有一个结构性的天花板——蒸馏出来的是一个"固定曲目"的策略。想加一个新技能？多数情况下得把它塞回专家池、重新蒸馏一遍。它把多技能"装"进了一个网络，却没给你"现场组合"的能力。上海交大与上海 AI Lab 的 EAGLE 用迭代式的"通才↔专才"循环去缓解跨机器人的重训（arXiv:2602.02960，2026 预印本，数据尚未经同行评议），但"加技能不重训"这件事，路线 A 本质上没解决。

路线 B：不合并技能，合并出一个"技能空间"

第二条路更聪明，也更学术。它不把技能蒸馏成一个"会做 N 件事的策略"，而是蒸馏成一个技能隐空间——一个低维连续的"技能地图"，每个坐标点对应一种可执行的动作风格。下游要用的时候，冻结这张地图，只训一个上层策略学会"在地图上挑点"，就能复用、串联、组合底层技能，而完全不动底层。

这一路的代表作是 UC Berkeley 与英伟达的 Xue Bin（Jason）Peng 做的 ASE。它用对抗式动作先验加上无监督的技能发现，从一大堆无标注的剑术动作里预训练出一个低维技能嵌入。之后要让角色学会"冲刺劈砍""格挡还击"，只需在这个空间上训个高层策略——底层的运动技能是现成的、可复用的。它的美感在于：技能一次学成、终身复用，下游任务像在调用一个技能库的 API。

ASE 之后，Chen Tessler 的 CALM（arXiv:2305.02195）给这个隐空间加上了语义可控性——你可以用一段风格描述直接"指挥"角色，可控性从 ASE 的 35% 提到 78%。而把这条路推到集大成的，是 CMU 罗正禹（Zhengyi Luo）的 PULSE：它先有一个能模仿上万条动作的单一控制器 PHC，再用一个带信息瓶颈的编码器，把 PHC 的全部本事蒸馏进一个 32 维的通用运动隐空间。这一步很关键——它是路线 A 和路线 B 的显式桥接：把"一个会做很多事的策略"（A 的产物）转化成"一张可复用的技能地图"（B 的形态）。蒸馏只损失了一点点保真度（成功率 99.2%→97.1%），换来的是下游 VR 全身追踪成功率从 ASE 的 37.6% 跃升到 93.4%。

路线 B 优雅得几乎无可挑剔——除了一个致命的现实：它几乎全部活在仿真里。ASE 的剑客、CALM 的角色、PULSE 的虚拟人，都是物理仿真中的数字角色，真机验证极少。可组合的技能空间是有了，但它还没走出屏幕。这就引出了整个领域最尴尬的一道裂缝，后面宏观观察里会专门说。

路线 C：不蒸馏，直接把独立策略"加"起来

前两条路都要一个"重新训练/蒸馏"的环节。而你问题里最理想的那种——拿别人单独学好的技能，不重训就拼起来——对应的是第三条路。这条路这两年出了几个相当漂亮的结果，机制各不相同。

最有想象力的是 MIT 王力锐（Lirui Wang）的 PoCo。它的洞察是：扩散策略本质上是在表示一个动作的概率分布，而概率分布是可以相乘、相加的。于是它把分别在仿真、真机、人类视频上、用 RGB、点云、触觉独立训练的多个扩散策略，在推理时把各自的去噪梯度（score）直接相加，就合成出一个复合策略——全程不训练任何新东西。效果很说明问题：单用仿真策略成功率 90%、单用真机策略 50%，两者组合后达到 100%。这是"免重训组合独立策略"目前最有 novelty、又真的上了真机（Franka + 触觉手）的样板。

沿着"score 相加"这条线，港大的 GPC 把它做成了测试时的凸加权组合（arXiv:2510.01068），在真机上再涨 10%。而如果你想要的是数学上可证明的组合，那要看南非 Witwatersrand 大学 Tasse 等人的布尔任务代数（NeurIPS 2020）：把任务组合形式化成布尔运算，学好一组基础技能后，对它们的价值函数做"与 / 或 / 非"运算，就能零样本得到新任务的最优策略——这是整个领域唯一有最优性保证的免重训组合，理论根基可追到 Haarnoja 2017 的可组合最大熵策略。代价是它的最优性依赖受限的动力学假设，几乎只在离散或受限仿真里验证过，连续控制上的"非"运算尤其脆弱。

还有两种更"工程"的拼法。MIT 同一个王力锐的 FLEET-MERGE（ICLR 2024）走的是参数合并：处理好循环网络神经元的排列不变性后，把在 50 个不同任务上分别训练的策略参数直接合并成一个，不需集中重训。而最贴近你踢球项目的，是清华的 RPG（arXiv:2604.21355，2026 预印本）：它把每个格斗技能独立训练并冻结，再训一个很轻的门控网络输出"专家动作的加权混合"（上下半身分开门控），在 Unitree G1 真机上把技能切换成功率从基线的 0.05–0.65 提到 0.70–0.95——和你的平台完全一致，是"冻结专家 + 轻量门控"复用已有独立策略的清晰案例。Oregon State 的搬箱工作（ICRA 2024）则更朴素：把搬箱拆成走 / 站 / 拾取 / 抱走 / 抱站五个独立策略，用状态机顺序切换拼完全程。

但这里要拆穿一个措辞游戏：RPG、搬箱这类"门控 + 状态机"，严格说是顺序调度，不是真正的组合——它保证技能之间切换平滑，却不产生"同时做两件事"的新行为。真正的"边走边操作"这种强耦合动作，现有的免重训方法大多还做不到，仍要联合训练或至少加一个残差层。而 score 组合虽然能"叠加"，也有硬前提：它本质是概率密度相乘，只有当各策略的动作分布部分重叠、互不矛盾时才近似正确；技能目标一冲突，组合就退化甚至失效。

路线 D：绕开组合，用大数据一次学完

最后是一条"耍赖"但极其有效的路：既然组合这么难，那就别组合了——用足够多的数据，把几百种技能一次性喂进一个大模型。这条路的地基是 Google DeepMind 牵头、21 家机构 34 个实验室合建的 Open X-Embodiment：把 60 个已有数据集汇成迄今最大的开源真机数据集，100 万+ 条轨迹、22 种机器人本体、527 种技能。它证明了一件颠覆性的事——把不同机器人的数据混在一起训，能跨本体正向迁移，互相涨点。

有了数据，就有了模型。Berkeley 的 Octo、Stanford/Berkeley 的 OpenVLA（7B 参数、970k 轨迹训练，arXiv:2406.09246，GitHub 6.6k star）把这批数据炼成开源的 VLA 模型：输入一张图和一句话，直接输出动作，一个模型跨物体、跨任务。而把这条路推向产品化的是英伟达 GEAR 组的 GR00T——号称首个开源人形基础模型，双系统架构（视觉语言慢思考 + 扩散动作快执行），一个模型跨本体做抓取、双臂搬运、多步操作。

路线 D 的技能不是"组合"出来的，是"一起学"出来的。它用数据和算力的蛮力，绕过了灾难性遗忘——因为所有技能从一开始就在同一个网络里共存。这也是为什么这条路由工业界领跑：Physical Intelligence 的 π0（开源实现 openpi 有 12.8k star）、Figure 的 Helix、Google 的 Gemini Robotics，背后拼的是私有数据规模和算力，而这恰恰是学界最缺的。

宏观观察：一道没弥合的裂缝，两个正在崛起的中心

最深的一道裂缝，在"真机"和"可组合"之间。把前三条路线叠在一起看，会发现一个别扭的事实：真机人形几乎全走路线 A（蒸馏进单网络，不可组合），而真正可组合的路线 B（技能隐空间）几乎全困在仿真里。你想要的"在真机上自由组合已学技能"，恰好落在两条成熟路线都覆盖不到的缝隙里。PULSE 把 A 和 B 在仿真里接通了，但没人把这套可组合隐空间成功搬上真机。这不是某个技术难点，而是整个领域的结构性空白——"真机上可组合的技能空间"至今没有成功范例。

方法谱系上，两条河正在汇流。一条是图形学的运动模仿血脉：NPMP（2019）→ ASE（2022）→ PULSE（2024），工程栈高度统一，几乎全是 Isaac Gym + AMASS 动捕 + SMPL 人形。另一条是机器人学习的大数据血脉：OXE（2023）→ OpenVLA（2024）→ GR00T（2025）。两条河都在往"一个模型承载多技能"的入海口汇，而站在交汇处、同时踩住两条线的，是英伟达——它既有 Isaac 仿真栈和 HOVER/ASE/CALM，又有 GR00T 和跨本体数据，是当前生态里最强的整合者。

还有一个不该忽略的信号：华人实验室的集体崛起。这份调研里，真机多技能蒸馏的关键工作——HOVER（何泰然）、PULSE/OmniH2O（罗正禹）、ExBody2（项晓龙组）、BumbleBee（北大卢宗青）、EAGLE（上交+上海 AI Lab）——几乎清一色由华人一作或华人实验室主导，且集中在 2024–2026 爆发。方法创新的重心，正从传统的 DeepMind 一家独大，向 NVIDIA + CMU/UCSD/Berkeley 华人学者 + 中国高校这个网络转移。

而在离产品最近的地方，工业界已经反超。whole-body 的方法学界仍领跑，但一到"多技能长程自主 + 真实部署"，Figure（Helix）、NVIDIA（GR00T）、Google（Gemini/Apollo）明显走在前面——代价是它们的成果大量以官方演示呈现、缺独立复现，可信度要打折。开源侧则出现了明确的汇聚点：Hugging Face 的 LeRobot（25.9k star）成了新项目的默认起点，Isaac Lab、ProtoMotions 是仿真训练的公共基座。

瓶颈与开放问题

  1. 真机 loco-manipulation 的精细度。现有真机蒸馏多是"全身模仿人体动作"，接触丰富、需要力控精度的操作（踢球受力、抓握）还没被纳入统一策略。会跳舞不等于会踢弧线球。
  2. A、B 两路线没融合。真机走不可组合的 A，可组合的 B 只在仿真——缺一个"真机上可组合隐空间"的成功范例，这是最值得攻的方向。
  3. 持续 / 增量蒸馏（灾难性遗忘）。PHC 的渐进式容量分配、EAGLE 的迭代蒸馏是初步方案，但"加一个新技能仍要整体重训"这件事没有根治。
  4. 免重训组合的强前提。score 相加要求技能分布不冲突；逻辑代数要求受限动力学；参数合并对网络架构敏感。三者都还不是"任意技能即插即用"。
  5. 跨本体泛化。一个策略真正零样本跨不同形态人形，EAGLE 刚起步，远未解决。
  6. 基准不统一。各家成功率 / MPJPE 口径、动作集、仿真器都不同，横向几乎不可比——HOVER 换个仿真器成功率从 63% 掉到 16%，就是这问题的缩影。

回到你的踢球项目

如果你打算走"分别学好走位与踢球，再组合"的路线，最贴近、最可复现的范式是清华的 RPG——同样是 Unitree G1，冻结独立训练的专家、只训一个轻量门控做平滑切换。它能让机器人在"走位模式"和"起脚模式"之间干净地切换，这一步现有技术已经够用。

但你真正想要的"边走位边实时调整踢球弧线"，属于强耦合的全身控制——支撑腿的平衡和摆动腿的击球在物理上互相牵制，"分别学再叠加"会违反动力学一致性，叠出来的动作可能根本站不稳。对这种情况，本期所有免重训组合方法大概率都不够，务实的做法是：用冻结的走位 / 踢球专家提供先验，再加一个轻量的残差层或联合微调去学两者耦合的那部分——既省去从零训练，又补上纯组合缺失的物理一致性。这几乎就是 HOVER "教师打底 + 蒸馏统一"和 RPG "冻结专家 + 门控"两种思路的结合。
